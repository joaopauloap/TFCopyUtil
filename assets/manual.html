Manual de Ajuda do TF Util

Introdução

O TF Util é um programa criado com o objetivo de otimizar a produtividade no desenvolvimento do Terminal Financeiro. Entre suas principais funcionalidades, destacam-se a capacidade de criar cópias e estruturas de pasta dos arquivos de saída compilados do Eclipse para a pasta CLIENT/localwkst do Terminal Financeiro, proporcionando atualizações mais rápidas do projeto. Além disso, o programa também disponibiliza funções para exportar o projeto para outros colaboradores ou fazer documentações, importar copy books para a implementação de fluxos do mainframe e oferece atalhos relacionados ao Terminal Financeiro.

Seção 1 - Trabalhando com a Local Workstation

A localwkst é uma pasta localizada em C:\CLIENT\localwkst onde são providas as classes .java compiladas (.class) e demais arquivos de saída do eclipse dos projetos do Terminal Financeiro (TF) durante o processo de desenvolvimento de atualizações do TF. Como a saída desses arquivos se dá na pasta workspaceFDDB do eclipse Bradesco, para efetivar as alterações no TF é necessário replicar toda a estrutura de pastas e seus arquivos modificados na localwkst. A primeira função do TF Util é auxiliar neste processo.

  1. O primeiro passo é informar os caminhos de arquivo na Área de caminhos (1). Cada linha corresponde a um arquivo. É possivel obter o caminho rapidamente dentro do eclipse com o comando ALT + Enter dentro do arquivo em que esteja editando.
  2. O que deve ser copiado para a localwkst são as classes compiladas .class e/ou outros arquivos da saída da compilação, porém ao copiar o caminho do arquivo pelo eclipse você irá obter o caminho do código fonte. A opção Interpolar saída java (2) localiza os arquivos de saída a partir dos do código fonte, desde que tenha ocorrido a construção (build) bem sucedida do projeto. A opção guardar caminhos mantem
  3. Por fim basta clicar no botão Transferir arquivos (3) para dar início ao processo de cópia dos arquivos. Mensagens de erro ou sucesso do processo serão exibidas.

Seção 2 - Exportando pastas

Para facilitar a exportação apenas dos arquivos modificados, existem estas duas opções, as quais quando selecionadas irão abrir uma tela solicitando para qual diretório deseja exportar as pastas. São opções uteis quando deseja-se compartilhar o projeto entre colaboradores ou realizar entregas/documentações, etc.

  1. Exportar localwkst
  2. Exportar workspaceFDDB

Seção 3 - Importando copybooks

Copybooks são arquivos que representam as estruturas de dados das tabelas do mainframe. São enviados pela equipe de mainframe, geralmente no formato .txt. Para implementar um fluxo no projeto do TF são necessários o arquivo .xml e .dsn, obtidos ao compilar as copybooks no projeto ARQ_PCCF pelo eclipse. Para que isso seja feito, as copybooks precisam estar no formato .cpy dentro de uma pasta com o nome do fluxo em:

C:\workspaceFDDB\ARQ_PCCF\resourcedata\cobolcopys\{Nome_do_Fluxo}

Também é necessário um arquivo .fwk com o nome do fluxo, onde são declaradas os nomes das copybooks de entrada e de saída. Este arquivo fica em:

C:\workspaceFDDB\ARQ_PCCF\JavaSource\resourcedata\fwk\

Após isso o projeto ARQ_PCCF deve ser compilado pelo eclipse e o xml e dsn ficarão disponíveis na pasta output.

O TF Util possui a opção Importar books para automatizar este processo. Ao ser selecionada, abrirá uma tela onde deverão ser selecionadas as copybooks. Multiseleção é permitida segurando CTRL, conforme ilustra a imagem abaixo. Após isso, o nome do fluxo deve ser escrito antes de cada caminho, separados por um espaço apenas. Por fim, basta clicar no botão Transferir arquivos.

Seção 4 - Atalhos e outras opções

Esta seção TF Util contém os atalhos mais utilizados pelos desenvolvedores que trabalham em projetos no terminal financeiro. Como ja mencionado anteriormente, a opção Interpolar saída java serve para que o TF Util consiga localizar os caminhos dos arquivos de saída a partir dos caminhos de fonte fornecidos. Já a opção Guardar caminhos mantém o histórico dos caminhos utilizados na área, permitindo que o programa seja fechado e reaberto sem perda desses caminhos.

Atalhos

  1. Abre a pasta localwkst
  2. Abre a pasta WorkspaceFDDB
  3. Abre o log (com o editor de texto definido nas Preferências)
  4. Encerra o processo do terminal financeiro
  5. Abre o terminal financeiro
